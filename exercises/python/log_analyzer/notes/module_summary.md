可以把这四个模块理解成四个不同的“变化边界”：

| 模块 | 当前职责 | 最可能因为什么需求变化而修改 |
|------|----------|------------------------------|
| `data_loader.py` | 读取 CSV、检查字段、验证数据 | 输入数据格式或数据校验规则变化 |
| `analyzer.py` | 根据有效记录计算分析结果 | 分析指标或业务规则变化 |
| `report.py` | 组织报告文本并写入文件 | 报告内容、展示格式或输出介质变化 |
| `main.py` | 处理命令行参数、组织执行流程 | 程序运行方式或流程编排变化 |

# 1. `data_loader.py`：输入数据需求变化

当前它负责：

- 读取 CSV
- 检查必需列：`experiment_name`、`train_loss`、`val_loss`、`accuracy`
- 检查数值类型
- 检查 `loss` 非负
- 检查 `accuracy` 在 `[0, 1]`
- 跳过无效行并记录错误
- 检查实验名称是否重复

因此，下面这些需求最可能修改 `data_loader.py`：

- CSV 增加字段，例如 `learning_rate`、`epoch`、`model_name`
- 输入从 CSV 改成 JSON、Excel 或数据库
- `accuracy` 改成百分数形式，例如 `85.6`，而不是 `0.856`
- 允许空的 `val_loss`
- 允许重复的实验名称
- 对错误行不再跳过，而是直接终止
- 增加日期、模型名称、数据集名称等字段校验

例如，如果输入格式改成 JSON，主要应该修改 `data_loader.py`，而不是 `analyzer.py`。

# 2. `analyzer.py`：分析规则变化

当前 `analyzer.py` 只做了两类分析：

- 找出 `accuracy` 最高的实验
- 计算平均准确率

因此，下面这些需求会修改它：

- 增加最低 `val_loss` 实验
- 计算准确率的最大值、最小值、中位数
- 按模型或数据集分组统计
- 找出训练损失下降最快的实验
- 将“最佳实验”从最高准确率改为最低验证损失
- 使用加权平均，而不是普通平均
- 对不同实验设置排名
- 增加实验之间的比较或趋势分析

例如：

> “最佳实验必须同时满足准确率最高且验证损失低于 0.2。”

这是分析规则变化，应放在 `analyzer.py`。

需要注意，当前 `main.py` 同时使用了 `ExperimentAnalyzer` 和 `analyze_experiments()`。其中：

> 得到的 `record` 当前没有被使用。因此，如果以后统一分析逻辑，可能需要调整 `main.py` 的调用方式，但“分析规则本身”仍然属于 `analyzer.py`。

# 3. `report.py`：报告需求变化

当前 `report.py` 负责：

- 把分析结果格式化成文本
- 将文本写入 `analysis_report.txt`

因此，下面这些需求会修改它：

- 报告增加标准差、最佳模型、错误详情
- 报告从英文改成中文
- 报告改成 Markdown、JSON、HTML 或 CSV
- 调整字段顺序和小数位数
- 将错误行详细列出，而不是只显示错误数量
- 同时输出控制台报告和文件报告
- 输出到数据库、API 或对象存储

例如：

> “输出一份适合网页展示的 HTML 报告。”

主要应修改 `report.py`，因为这是展示和输出格式的变化，不是数据读取或分析逻辑的变化。

# 4. `main.py`：运行方式和流程变化

当前 `main.py` 负责：

- 定义命令行参数
- 配置日志
- 调用数据加载、分析和报告模块
- 处理异常
- 返回退出码
- 打印最终报告

因此，下面这些需求会修改它：

- 增加 `--format json` 参数
- 增加 `--strict` 参数，决定遇到无效行时是否终止
- 支持一次处理多个输入文件
- 增加 `--quiet`、`--verbose` 等命令行选项
- 改成定时任务或 Web API 的入口
- 增加配置文件
- 改变模块调用顺序
- 将分析结果发送到邮件、消息队列或远程服务
- 修改异常处理和退出码规则

例如：

> “用户可以通过命令行选择输出 Markdown 还是 JSON。”

这通常需要修改 `main.py` 来增加参数，同时修改 `report.py` 来实现对应的格式化逻辑。

# 快速判断修改位置

可以用一个判断方法快速定位修改位置：

| 如果需求涉及…… | 应修改的模块 |
|----------------|--------------|
| 输入长什么样？ | `data_loader.py` |
| 怎么算结果？ | `analyzer.py` |
| 结果怎么展示或保存？ | `report.py` |
| 用户怎么启动和控制程序？ | `main.py` |

这种划分的价值在于：

- CSV 格式变化时，不必改分析算法。
- 分析指标变化时，不必改文件读取。
- 报告样式变化时，也不必改业务计算。
